Overview

This site contains examples of accessible navigation menus.

Caveats

When reviewing these pages, please keep the following aspects in mind:

  • These menus have been optimized for accessibility. Some further testing with other accessibility experts and assistive technology users is planned, so some features may change.
  • The site contents are currently dynamically generated to provide placeholder text. It is fairly easy to change the pages and headings on a page in case more relevant categories are needed.
  • The pages and menus are not currently designed to be responsive as of yet. Performance on smaller screens will have issues.
  • Many stylistic aspects of the menus can be changed. This includes colors, spacing, icons, etc. Current colors do follow SU branding while allowing for accessibility.
  • These examples are for primary site navigation, meaning the main navigation typically at the top of a web page. The placement and styling of any secondary navigation can be discussed. The current secondary navigation seen in the right column of some pages is merely a placeholder for now.
  • All pages do have Breadcrumb navigation when appropriate. This is recommended for accessibility purposes.

Menus

For each menu type, a brief description is provided including interaction features. Benefits and drawbacks are also provided.

Flat Menu

The Flat Menu is the most basic navigation menu. The navigation only contains links to the top-most pages.

Features, Benefits, and Drawbacks

Features

None

Benefits

  • Simple implementation.
  • No accessibility fixes needed beyond proper semantic HTML.
  • Does not require JavaScript and can work on older browsers.

Drawbacks

  • Requires every top-level page, if appropriate, to have a sub-navigation menu.
  • Users cannot discover site structure without clicking.
  • Users must rely on memory of site structure or site search.
  • Desire to improve discoverability of site often leads to more links in the menu, which makes it harder to support in smaller screens.

Dropdown-Only Menu

The Dropdown-Only Menu uses a dropdown/disclosure widget to show a sub-menu of pages. This variant places all links (save Home) under a submenu triggered by a button.

Features, Benefits, and Drawbacks

Features

  • Meets expected interaction options for accessiblity:
    • Toggle button can be triggered by touch, click, space key, enter key, or ↓ (down arrow) key.
    • Once the menu is open, users can navigate the submenu via tab, ↑ (up arrow) key, and ↓ (down arrow) key.
    • An open menu can be closed by the Esc (escape) key. Focus is returned to the menu's toggle button.
  • Menu closes when the user clicks/touches anywhere outside of the menu widget. This is optional and can be removed (not a requirement for accessibility but common).
  • The top-level link is the first link in the submenu.

Benefits

  • Improved discoverability of site structure.
  • Uses basic HTML, CSS, JavaScript, and ARIA markup supported by all modern browsers.
  • Target (i.e., button) sizes are large for easy accessibility.

Drawbacks

  • To get to a top level page, one has to toggle open the menu.
  • Users must understand that the down triangle icon means it is a menu toggle button and not a link.
  • Users might click on button expecting a link.
  • Requires JavaScript and may not work on older browsers although the menu will default to a basic flat menu (progressive enhancement).

Dropdown-Link Menu

The Dropdown-Link Menu uses a dropdown/disclosure widget to show a sub-menu of pages. This variant keeps the top-level item as a link and then provides a toggle button to display a menu of the sub-pages.

Features, Benefits, and Drawbacks

Features

  • Meets expected interaction options for accessiblity:
    • Toggle button can be triggered by touch, click, space key, enter key, or ↓ (down arrow) key.
    • Once the menu is open, users can navigate the submenu via tab, ↑ (up arrow) key, and ↓ (down arrow) key.
    • An open menu can be closed by the Esc (escape) key. Focus is returned to the menu's toggle button.
  • Menu closes when the user clicks/touches anywhere outside of the menu widget. This is optional and can be removed (not a requirement for accessibility but common).
  • The top-level link is the first link in the submenu.

Benefits

  • Improved discoverability of site structure.
  • Uses basic HTML, CSS, JavaScript, and ARIA markup supported by all modern browsers.
  • Users do not have to go to the top-level page.

Drawbacks

  • Adds an extra tab stop (the toggle button) for each top-level link.
  • Users must understand that the down triangle icon means it is a menu toggle button and not a link.
  • For accessibility, the toggle button needs to be of a minimal size (ideally a little larger than it is currently).
  • Requires JavaScript and may not work on older browsers although the menu will default to a basic flat menu (progressive enhancement).

Code

The code for theses individual menu schemes were written to be fairly barebones and independent of existing libraries. The individual menu options will have their code separated into distinct codebases in the near future.

Site Generation

Each menu variant and its pages are generated dynamically.

  • The overall site structure is defined in site.yaml, a human-readable YAML file that defines top-level pages, subpages, and H2-level headings on each page.
  • Distinct pages are defined by the URL query parameters page and subpage as defined in the YAML file.
  • Randomized text content was gathered from Wikipedia using this placeholder text generator.
  • To guarantee the same content appears on each page, a seed query parameter is used to prime a pseudorandom number generator. The seed is a hash of the concatenation of the page and subpage params.
